# flight-intel refactor — 2026-05-25

## Objetivo (Elias, voice 2026-05-25)

Eliminar el mapa global de aeropuertos / route registry y enfocar flight-intel en **tres scrapers** que ya tenían sentido + un **dashboard de rutas óptimas para fechas pre-definidas**.

## Scope

### Fuentes (única fuente de verdad de precios/rutas)
1. **Google Flights** — modo multi-destino (multi-city). Devuelve combinaciones con link de compra del pack completo cuando alguna OTA lo vende.
2. **Booking Flights** — modo multi-destino. Mismo formato, link del pack si disponible.
3. **Kayak** — multi-city via Playwright. Retry agresivo hasta que cargue (UA/cookies/headful fallback).
4. **Per-airline fallback** — cuando GF/Booking/Kayak optimizan ruta pero NO entregan link de compra del pack (porque ninguna OTA lo vende), abrir las páginas oficiales de las aerolíneas de cada tramo y traer el precio + link directo del tramo individual. Totalizar.

### Dashboard
Una página por viaje pre-definido (fechas de `flight-monitor/patterns.py` → `FIXED_TRIPS` + `PATTERNS A/B`):

Para cada viaje:
- **Ruta óptima** (o top-N) propuesta por cada fuente.
- **Link directo de compra del pack** (Google Flights / Booking / Kayak) cuando esté disponible.
- **Cuando el pack no tiene link**: listar cada tramo con su link directo de aerolínea + **totalizador del valor**.
- Comparativa precio total: GF pack vs Booking pack vs Kayak pack vs Σ tramos independientes.

### Fechas pre-definidas (importadas de flight-monitor)
- SCL↔GRU 18→29 jun 2026 (FIXED_TRIPS GRU-2026-06)
- SCL↔GRU varios jun 2026 (3→7, 3→8, 4→7, 4→8, 4→9, 5→7, 5→8, 5→9)
- SCL↔FLN 29-oct→2-nov 2026
- GIG↔FLN 29-oct→2-nov 2026
- Patrón A: SCL↔GIG jue→dom 10d (rolling, ≥14h)
- Patrón B: SCL↔GIG mié→lun 12d (rolling, ida ≥18h, vuelta ≥14h)
